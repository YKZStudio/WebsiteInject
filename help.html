WebsiteInject 使用说明

导入脚本，对指定网站进行注入与修改。本页讲怎么写脚本、各项设置的含义，以及可用的 GM API。

工作原理 写第一个脚本 匹配网站 注入时机 注入世界 元数据头 GM API 导入 / 导出 / 备份 常见问题

工作原理

本扩展基于 Chrome 官方的 chrome.userScripts API。你保存的每个脚本会按你设定的「匹配网站」自动注入到对应页面执行，可以改 DOM、加样式、hook 页面 JS。脚本和设置存在浏览器本地（chrome.storage.local），不上传任何服务器。

写第一个脚本

脚本就是一段普通 JavaScript，注入后在目标页面里运行。点管理页的 ＋ 新建 / 粘贴脚本，填好「匹配网站」，把代码粘进去即可。最简单的例子：

// 把页面背景改成黑色
document.body.style.background = "#000";
console.log("脚本已运行，当前地址：", location.href);

推荐用 IIFE 包起来，避免变量污染页面：

(function () {
  "use strict";
  // 你的代码
})();

匹配网站（match pattern）

每行一个规则，格式为 <协议>://<主机>/<路径>，任一规则命中就注入。

规则含义
*://*.example.com/*example.com 及其所有子域，任意路径（最常用）
https://example.com/*仅 https 的 example.com 主域
*://example.com/news/*只在 /news/ 下的页面
<all_urls>所有页面（慎用）

* 在主机里只能作开头的整段通配（*.），在路径里是任意字符。小窗里点 ＋ 为本站点新建脚本 会自动填好当前域名的规则。

注入时机（run-at）

值时机适合
document_start页面刚开始加载、DOM 还没建抢先 hook fetch、屏蔽脚本、改全局变量
document_endDOM 构建完成（图片等资源可能还在加载）改页面结构、删元素
document_idle（默认）页面基本加载完、空闲时大多数场景，最稳妥

注入世界：USER_SCRIPT vs MAIN

USER_SCRIPT（默认，隔离）MAIN（页面主世界）
能改 DOM / 加样式✅✅
访问页面的 JS 变量 / 函数❌ 隔离，看不到✅ 直接访问、可 hook
改写页面的 fetch / XMLHttpRequest❌✅
安全性 / 与页面隔离高低（与页面共享环境）

简单说：只改外观或 DOM → 用 USER_SCRIPT（更安全）；要拦截网络请求、调用或改写页面自身的 JS → 用 MAIN。例如在 MAIN 世界里 hook fetch：

const orig = window.fetch;
window.fetch = function (...args) {
  console.log("请求：", args[0]);
  return orig.apply(this, args);
};

元数据头（可选）

脚本开头可以写一段油猴风格的元数据头。导入 .user.js 文件时，会自动解析其中的 @name / @match / @run-at 来预填表单：

// ==UserScript==
// @name         去除某站广告
// @match        *://*.example.com/*
// @run-at       document_idle
// ==/UserScript==

/* 你的代码 */

注意：元数据头只用于导入时预填。脚本保存后，实际生效的是表单里的「匹配网站 / 时机 / 世界」设置，而非头里的注释。

GM API

脚本里可直接调用以下 API，无需 @grant。完整演示见示例脚本 examples/gm-demo.user.js。

API说明
GM_setValue(k, v) / GM_getValue(k, def)
GM_deleteValue(k) / GM_listValues()		持久存储，按脚本分命名空间。读取同步、写入异步落盘，跨刷新保留。
GM_xmlhttpRequest(details)跨域请求（由后台发出，绕过页面同源限制）。支持 onload/onerror/ontimeout、method/headers/data/timeout、responseType:"json"。
GM_addStyle(css)注入一段 CSS。
GM_setClipboard(text)写入剪贴板。
GM_openInTab(url, background?)新标签页打开 URL。
GM_notification(text, title?)弹系统通知。
GM_info / GM_log(...)脚本元信息 / 控制台输出。
GM.*上述的 Promise 版本，如 await GM.getValue(k)、GM.xmlHttpRequest(...)。
unsafeWindow页面 window（仅 MAIN 世界为页面真实 window）。

例：持久计数

const n = (GM_getValue("count", 0) || 0) + 1;
GM_setValue("count", n);
console.log("第", n, "次运行");

例：跨域请求

GM_xmlhttpRequest({
  method: "GET",
  url: "https://httpbin.org/get",
  responseType: "json",
  onload: (res) => console.log(res.status, res.response),
  onerror: (e) => console.warn("失败", e.error)
});

导入 / 导出 / 备份

  • 导入 .js 文件：把本地脚本文件加进来（可多选）。
  • ⬇（单条）：把某个脚本导出为 .user.js 文件，可分享或导入到油猴。
  • 导出备份：把所有脚本 + GM 存储导出成一个 JSON 文件。
  • 导入备份：从 JSON 恢复，按脚本 id 合并（同 id 覆盖，不会产生重复）。

常见问题

改了开关 / 脚本，为什么没立刻生效？

注入的变更只对下一次页面加载生效。改完刷新一下当前页即可（小窗在切换开关后会提示并提供「刷新本页」按钮）。

脚本完全没运行？

① 确认已在 chrome://extensions 开启开发者模式，并在本扩展「详情」里打开「允许用户脚本」；② 确认「匹配网站」规则能命中当前地址；③ 打开 DevTools 控制台看有没有报错。

GM_getValue 在另一个标签页读到旧值？

为兼容同步读取，注册时会把已存的值「烘焙」成快照。同一标签页内写入即时生效；跨标签页的并发写入会在下次注册时刷新。需要实时值时用 await GM.getValue(k)。

MAIN 世界安全吗？

MAIN 世界与页面共享运行环境，隔离性较弱。在不完全信任的页面上，谨慎使用 MAIN 世界 + GM_xmlhttpRequest。
